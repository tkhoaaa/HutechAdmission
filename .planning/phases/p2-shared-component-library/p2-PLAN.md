# Phase 2 Plan — Shared Component Library

> Generated by Claude Opus 4.6 based on comprehensive component audit.

## Scope

Xây dựng lại 16+ UI components theo shadcn/ui pattern, đảm bảo:
1. Tất cả components có dark mode support
2. Export đầy đủ qua `index.js`
3. Không có duplicate/orphaned files
4. CSS tokens nhất quán

## Audit Summary

| Category | Count | Status |
|----------|-------|--------|
| shadcn-style (good) | 20+ | Well implemented via @base-ui/react |
| Custom legacy (needs work) | 11 | Various issues |
| Deprecated/orphaned | 2 | Delete |
| Missing exports | 23+ | Add to index.js |
| Dark mode broken | 3 | HIGH priority fix |

## Plans

### Plan 2.1: Index Exports Fix (MEDIUM)

**Problem:** `index.js` exports only 12 items, but 35+ components exist.

**Goal:** Export all components consistently from `index.js`.

**Acceptance Criteria:**
- [ ] Every component in `src/components/ui/` is accessible via `@/components/ui/`
- [ ] Named exports match component names
- [ ] No circular dependencies

**Files touched:** `src/components/ui/index.js`

---

### Plan 2.2: High-Priority Refactoring (HIGH)

**Problem:** Alert, Modal, FormField have broken dark mode and invalid Tailwind colors.

**Goal:** Refactor these 3 critical components to be theme-aware.

**Acceptance Criteria:**
- [ ] Alert has `destructive`, `default`, `secondary` variants using CSS vars
- [ ] Modal uses `bg-background`, `text-foreground`, `border-border`
- [ ] FormField replaces `text-error-*`, `text-success-*`, `text-warning-*` with semantic tokens
- [ ] All components support dark mode via `.dark` class

**Files touched:** `src/components/ui/Alert.jsx`, `src/components/ui/Modal.jsx`, `src/components/ui/FormField.jsx`

---

### Plan 2.3: Cleanup Duplicate/Orphaned Files (LOW)

**Problem:** `Card.modern.jsx` (orphaned), dual FeatureCard, dual skeleton.

**Goal:** Remove dead code and consolidate duplicates.

**Acceptance Criteria:**
- [ ] `Card.modern.jsx` deleted
- [ ] Only one skeleton implementation remains (shadcn `skeleton.jsx`)
- [ ] `Loading.jsx` removes redundant skeleton exports
- [ ] `FeatureCard` consolidated to single location

**Files deleted:** `src/components/ui/Card.modern.jsx`
**Files modified:** `src/components/ui/Loading.jsx`

---

### Plan 2.4: Low-Priority Polish (LOW)

**Problem:** Hero uses undefined `bg-grid-pattern`, dual toast systems.

**Goal:** Clean up remaining issues.

**Acceptance Criteria:**
- [ ] `Hero.jsx` removes or replaces `bg-grid-pattern`
- [ ] Only one toast system (prefer `sonner` over `react-hot-toast`)
- [ ] Stats, FeatureCard migrated to use CSS vars where possible
- [ ] `Grid.jsx` exports `GridItem`

**Files touched:** `src/components/ui/Hero.jsx`, `src/components/ui/Stats.jsx`, `src/components/ui/FeatureCard.jsx`, `src/components/ui/Grid.jsx`, `src/App.jsx`

---
